# 01. 前端 UI 与交互需求

> 来源：用户需求 §1, §2, §3, §4, §5, §8

## 1.1 页面清单（MVP）

> 来源决策：内网部署，无登录页（见 [06-auth.md](06-auth.md)）。

| 页面 | 路径 | 主要功能 |
|------|------|----------|
| 视频库首页 | `/` | 缩略图网格、搜索栏、新增下载、批量删除 |
| 下载任务页 | `/downloads` | 任务列表与实时进度 |
| 视频详情/播放页 | `/watch/:id` | 内嵌播放器、播放历史、相关操作 |

## 1.2 核心交互流

### 1.2.1 添加下载任务（需求 §2, §8）

```
[首页] → [点 "+ 添加下载" 按钮]
      → 弹出 "新增下载" 对话框
        - 字段 1：YouTube URL（必填，支持单视频/歌单 URL）
        - 字段 2：格式：视频（默认且仅此选项，裁切仅音频）
        - 字段 3：清晰度（最高画质 / 1080p / 720p / 480p / 最低）
        - 字段 4：保存目录（默认 `data/videos/`，可选手动指定）
      → [点 "开始下载" 按钮]
      → 前端先调用 POST /api/downloads/check 检测是否重复
         ├─ 不重复 → 直接 POST /api/downloads 入队
         └─ 重复 → 弹出"覆盖确认"对话框（详见 [03-library.md §3.4.1](03-library.md)）
              ├─ 用户取消 → 不发起下载
              └─ 用户确认 → POST /api/downloads { overwrite: true }
      → 立即在下载任务页显示新任务（状态：Queued → Downloading）
```

> **歌单批量下载**：对话框中勾选"按歌单批量下载"时，整组任务一次性入队，**串行**下载（见 [02-downloader.md §2.2.3](02-downloader.md)）。任务列表**扁平展示**，所有子任务与单视频任务混合在同一列表中（每行独立的状态/进度），不分组、不嵌套展开。

### 1.2.2 下载进度展示（需求 §2）

- **位置**：下载任务页
- **展示方式**：每个任务一行
  - 缩略图（来自 YouTube，详情见 [03-library.md](03-library.md)）
  - 视频标题
  - 进度条（百分比 + 实时速度 + 已下载/总大小）
  - 状态标签（Pending / Downloading / Merging / Ready / Failed）
  - 操作按钮（取消 / 重试 / 删除）
- **更新机制**：
  - 优先使用 **SSE（Server-Sent Events）** 推送进度
  - 备选：前端 2 秒轮询 `GET /api/downloads/{id}/status`
- **取消支持**：用户在下载中可点击"取消"，后端中断 yt-dlp 进程并清理临时文件

### 1.2.3 视频库展示（需求 §3）

- **布局**：响应式网格，每行 4-6 张缩略图（响应式：手机 2 列、平板 3 列、桌面 5 列）
- **每张卡片显示**：
  - 缩略图（从 YouTube 获取的封面）
  - 视频标题（最多 2 行截断）
  - 上传者（频道名）
  - 时长（mm:ss 或 hh:mm:ss）
  - 添加日期
  - 角标：未播放过 / 已观看 / 已看完
  - **多选 Checkbox**（hover 时显示，批量删除用）
  - **删除按钮**（hover 时右上角显示，单删用）
- **顶部工具栏**：
  - 搜索框（按标题、上传者）
  - 排序下拉框
  - **批量操作栏**（选中 ≥1 项时浮现）：
    - 已选数量："已选 N 项"
    - 按钮：[批量删除] [清空选择]
- **搜索与筛选**：
  - 顶部搜索栏（按标题、上传者搜索）
  - 标签筛选（按上传者、按时长区间、按状态）
  - **排序（默认：最新添加优先，已确认 ✅）**：默认按 `created_at DESC`，可切换为"最早添加 / 标题 A-Z / 时长"
- **分页**：滚动加载或翻页，每页 24 条

### 1.2.3.1 删除视频 UI（已确认 ✅）

#### 单个删除

```
[hover 卡片] → 右上角出现 [🗑] 按钮
   │
   ▼ 点 [🗑]
弹出确认对话框：
┌──────────────────────────────────────────┐
│ ⚠️ 确认删除                                  │
│                                            │
│ 即将删除以下视频：                          │
│  • Python 教程 [dQw4w9WgXcQ]               │
│                                            │
│ 文件大小：256 MB                            │
│ 播放历史也将一并删除。                      │
│                                            │
│              [取消]  [确认删除]              │
└──────────────────────────────────────────┘
   │
   ├─ [取消] → 关闭对话框
   └─ [确认删除] → DELETE /api/videos/{id}
                    → 成功后从列表移除该卡片 + Toast "已删除"
```

#### 批量删除

```
[选中 N 个卡片] → 顶部出现批量操作栏
   │
   ▼ 点 [批量删除]
弹出确认对话框：
┌──────────────────────────────────────────┐
│ ⚠️ 确认批量删除                            │
│                                            │
│ 即将删除以下 N 个视频：                     │
│  • Python 教程 [xxx] (256 MB)              │
│  • Another Video [yyy] (128 MB)            │
│  • ...（超过 5 个时折叠，底部显示 "..."）   │
│                                            │
│ 共计：N 个视频，X.XX GB 磁盘空间            │
│ 播放历史也将一并删除。                      │
│                                            │
│              [取消]  [确认批量删除]          │
└──────────────────────────────────────────┘
   │
   └─ [确认批量删除] → POST /api/videos/batch-delete { ids: [...] }
                       → 进度条反馈：删除中 X/N
                       → 完成后从列表移除选中卡片 + Toast "已删除 N 个视频"
```

#### API 行为

- 单删：`DELETE /api/videos/{id}` → 同时删除本地文件 + 数据库记录 + 关联历史（SQL `ON DELETE CASCADE`）
- 批量删：`POST /api/videos/batch-delete { ids: [1, 2, 3] }` → 后端循环处理，任一失败不影响其他
- 前端在删除中禁用其他操作按钮，避免重复提交
- 失败时显示具体错误（如文件被占用），部分成功的需逐项汇报

### 1.2.4 在线播放（需求 §4）

- **入口**：点击视频库任一卡片 → 跳转 `/watch/:id`
- **播放器特性**（强制）：
  - ▶️ 播放 / ⏸ 暂停（空格键快捷键）
  - ⏪ 后退 10 秒 / ⏩ 前进 10 秒（左右方向键）
  - 进度条：可拖拽调整进度（点击或拖动）
  - 当前时间 / 总时长 显示
  - 倍速播放（0.5x / 0.75x / 1.0x / 1.25x / 1.5x / 2.0x）
  - 音量控制
  - 全屏
  - **进度记忆**：拖动、关闭页面后再次进入，恢复到上次离开的位置（±2 秒内）
- **进度上报**：每 5 秒向后端 `PATCH /api/videos/{id}/progress` 上报一次当前进度

### 1.2.5 播放历史（需求 §5）

- **入口**：视频库首页 → 切换 Tab 到"最近观看" 或 通过侧边栏"历史记录"进入
- **每条历史记录展示**：
  - 缩略图
  - 视频标题
  - 上次观看时间（"3 小时前" / "昨天"）
  - 当前观看进度（百分比 + 进度条）
  - 按钮：[继续观看] [从历史移除]
- **历史清理**：设置页提供"清空历史"按钮（可勾选"仅清除 30 天前的"）

## 1.3 视觉与样式

- **样式方案**：**Vanilla CSS**（不使用 TailwindCSS，除非用户后续明确要求）
- **设计风格**：现代极简，深色模式优先（YouTube 风格），可切换浅色
- **图标**：使用 lucide-react 或类似开源图标库
- **字体**：系统默认字体栈（-apple-system, Segoe UI, Roboto）
- **响应式断点**：移动端 < 768px、平板 768-1024px、桌面 > 1024px

## 1.4 前端技术选型（已确认 ✅）

- **技术栈**：**React 18 + Vite + TypeScript + Vanilla CSS**
- **优点**：
  - Vite 提供极速热更新，开发体验极佳。
  - TypeScript 保证参数类型安全（特别是视频对象和进度状态）。
  - Vanilla CSS 保证样式极致轻量，且没有 Tailwind 带来的标签堆砌。

## 1.5 范围裁切说明（已确认 ✅）

> 根据 2026-07-07 决策，以下 UI 组件或页面已被明确**裁切（不做）**：

- **登录页**：内网部署，直连可用，彻底移除无认证页面。
- **播放列表页（Playlist）**：不支持自建播放列表，移除侧边栏"播放列表"入口。
- **视频编辑页**：不支持标题、描述、标签的编辑。
- **设置页（简化）**：
  - 磁盘空间：展示 `data/` 目录的总大小及系统剩余磁盘空间（饼图）。
  - **yt-dlp Cookies 配置**：
    - 支持上传 `.txt` 格式的 Cookie 文件（支持 Netscape 格式，用于越过 YouTube 年龄限制或防 Ban）。
    - 按钮：[选择并上传 cookies.txt]（POST `/api/settings/cookies`）。
    - 状态展示：已上传 Cookie（显示上传时间及过期日期） / 未上传。
    - 按钮：[清除当前 Cookies]。
  - **下载代理配置（已确认 ✅）**：
    - yt-dlp 所有下载请求必须通过用户配置的代理服务器（用于翻越 GFW / 解决本地出口 IP 被封）。
    - **开关**：启用 / 禁用代理下载（默认启用）。
    - **协议选择**：HTTP / HTTPS / SOCKS5（默认 HTTP）。
    - **代理地址**：host（必填）。
    - **代理端口**：port（必填）。
    - **用户名 / 密码**（可选，需要鉴权的代理才填）。
    - **按钮**：[测试连通性]（后端用代理访问 youtube.com，返回成功/失败 + 耗时）。
    - **按钮**：[保存配置]。
    - 后端接口：`GET /api/settings/proxy`、`PUT /api/settings/proxy`、`POST /api/settings/proxy/test`。
  - **历史清理**：提供"立即清理 30 天前的"按钮。
  - **系统日志**：展示最近 50 条 logs，便于调试。
